# 静的UIデモ

Saleslog は顧客情報、活動記録、Agent実行履歴、承認操作を扱うため、公開環境では本物のDB保存や外部LLM接続を行わない構成を基本にする。

`frontend` は `VITE_DEMO_MODE=static` のとき、API通信を `frontend/src/api/staticDemo.ts` の合成データ応答に差し替える。画面、ルーティング、フォーム、Agent承認UIは通常のフロントエンド実装を使い、外部APIへの送信だけを避ける。

## できること

- ダッシュボード、顧客一覧、顧客詳細、活動記録、エリア別ボード、ユーザー管理の画面遷移
- 顧客や活動記録の登録・更新・削除の画面操作
- Agentの提案、根拠表示、編集、承認、却下の画面操作

## 行わないこと

- Clerk による認証
- バックエンドAPIへの送信
- DBへの保存
- 外部LLMへの送信
- 実データの表示

保存操作はブラウザ上の合成データだけを更新する。画面上部には「静的UIデモ」であることを常時表示する。

## ビルド

```bash
cd frontend
npm run build:static-demo
```

Cloudflare Pages などで公開する場合は、ビルドコマンドに `npm run build:static-demo` を指定し、出力ディレクトリを `frontend/dist` にする。

## 通常実行との違い

| 項目 | 通常実行 | 静的UIデモ |
|---|---|---|
| フロントエンド | 同じ | 同じ |
| API | FastAPI | 合成データ応答 |
| 認証 | Clerk またはローカル固定ユーザー | 合成ユーザー |
| DB保存 | あり | なし |
| Agent | バックエンドで実行 | 合成応答 |
| 外部LLM | 設定時のみ使用 | 使用しない |
