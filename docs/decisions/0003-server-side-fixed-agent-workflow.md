# ADR-0003: Agentフローをサーバ側固定にする

## ステータス
採用済み

## 背景

営業支援CRMでは、顧客情報、活動履歴、社内ナレッジの参照範囲をサーバ側で制御する必要がある。自由なツール選択にすると、認可境界の検査が難しくなる。

## 検討した選択肢

| 選択肢 | 利点 | 欠点 | 採用しなかった理由 |
| --- | --- | --- | --- |
| 自律的なツール選択 | 柔軟 | 認可・監査・再現性の境界が曖昧になる | CRMのデータ範囲に合わない |
| サーバ側固定フロー | 検証しやすい | 実行順序の自由度は低い | 認可とテストを明確にできる |

## 決定

Agent runはサーバ側の固定フローで進める。LLM相当の処理は文章生成と分類に限定し、source取得、citation確定、approval状態遷移、認可、DB保存はサーバ側で行う。

## 検証

| 方法 | 対象 | 結果 |
| --- | --- | --- |
| `backend/.venv/Scripts/python.exe -m pytest` | Agent run作成、権限外404、イベント保存内容、source ACL再確認 | `docs/verification.md` に backend pytest の通過結果を記録 |

## 影響

固定フローにより、評価ケースを追加しやすい。複数Agentの協調や自由なツール実行は扱わない。
